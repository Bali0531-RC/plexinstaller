üìñ Advanced Server Configuration Guide

For experienced system administrators who need detailed control over the installation process:

Database Configuration

MySQL/MariaDB Setup:

# Create database and user
mysql -u root -p -e "CREATE DATABASE plexapp_db;"
mysql -u root -p -e "CREATE USER 'plexapp'@'localhost' IDENTIFIED BY 'secure_password';"
mysql -u root -p -e "GRANT ALL PRIVILEGES ON plexapp_db.* TO 'plexapp'@'localhost';"

Nginx Configuration Examples

Reverse Proxy Configuration:

server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

Systemd Service Management

Service Control Commands:

# Start/stop services
systemctl start plexapp
systemctl stop plexapp
systemctl restart plexapp
systemctl status plexapp

# Enable/disable auto-start
systemctl enable plexapp
systemctl disable plexapp

# View logs
journalctl -u plexapp -f

üîç Troubleshooting Common Issues

Port Already in Use

If you encounter "port already in use" errors:

# Find process using port 80/443
sudo netstat -tulpn | grep :80
sudo netstat -tulpn | grep :443

# Kill process if necessary
sudo kill -9 <PID>

# Check for Apache/other web servers
sudo systemctl stop apache2
sudo systemctl disable apache2

SSL Certificate Issues

For Let's Encrypt certificate problems:

# Check certificate status
certbot certificates

# Renew certificates
certbot renew --dry-run
certbot renew

# Force certificate renewal
certbot certonly --force-renewal -d your-domain.com

Database Connection Errors

For database connectivity issues:

# Test database connection
mysql -u plexapp -p plexapp_db

# Check MySQL service
systemctl status mysql
systemctl status mariadb

# View database logs
tail -f /var/log/mysql/error.log

Permission Issues

Fix common file permission problems:

# Set correct ownership
sudo chown -R www-data:www-data /var/www/html
sudo chown -R plexapp:plexapp /opt/plexapp

# Set correct permissions
sudo chmod -R 755 /var/www/html
sudo chmod -R 644 /var/www/html/*.html
sudo chmod +x /opt/plexapp/start.sh
