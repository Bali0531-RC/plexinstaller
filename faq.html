General Questions

What is PlexDev.live? +

PlexDev.live is an unofficial community-created website that provides easy-to-use installation scripts for PlexDevelopment products. We are not affiliated with or endorsed by PlexDevelopment - we're just community members who wanted to make the installation process easier.

Important: This is not an official PlexDevelopment service!

Is this safe to use? +

Our scripts are designed to be safe and follow best practices, but as with any installation script:

  • Always backup your server before running any installation scripts
  • Review the script code if you're concerned about security
  • Test on a non-production server first if possible
  • We provide the scripts "as-is" without warranties

Good practices: Our scripts include error handling, rollback mechanisms, and detailed logging.

What's the difference between Stable and Beta versions? +

Stable Version:

  • Tested and verified to work reliably
  • Supports: PlexTickets, PlexStaff, PlexStatus, PlexStore, PlexForms
  • Recommended for production use

Beta Version:

  • Latest features and improvements
  • Includes all stable products PLUS: PlexLinks, PlexPaste
  • May have bugs or incomplete features
  • Good for testing new functionality

Installation Questions

What are the system requirements? +

Supported Operating Systems:

  • Ubuntu 18.04+ (recommended)
  • Debian 9+
  • CentOS 7+
  • Fedora 30+
  • Arch Linux
  • openSUSE

Minimum Requirements:

  • 512MB RAM (1GB+ recommended)
  • 10GB free disk space
  • Root or sudo access
  • Internet connection
Do I need a domain name? +

It depends on the product:

  • Web-based products (PlexStore, PlexForms, etc.): Yes, you need a domain or subdomain
  • Bot-only products (PlexTickets without dashboard): No domain required

For web-based products, the script will:

  • Set up Nginx reverse proxy
  • Configure SSL certificates automatically
  • Handle HTTPS redirects
How do I prepare my product files? +

Before running the installer:

  1. Download your PlexDevelopment product (ZIP or RAR file)
  2. Upload it to your server (any directory is fine - the installer will find it)
  3. The installer searches in: /home, /root, /tmp, and current directory
  4. If the installer can't find your file, it will ask for the full path

Tip: The installer prioritizes unobfuscated versions over obfuscated ones automatically.

What happens during installation? +

The installer performs these steps:

  1. System Detection: Identifies your OS and package manager
  2. Dependencies: Installs Node.js, Nginx, Certbot, and other tools
  3. File Extraction: Extracts your product files to /var/www/plex/
  4. NPM Install: Installs Node.js dependencies
  5. Network Setup: Configures ports and firewall
  6. Web Server: Sets up Nginx reverse proxy (if needed)
  7. SSL Certificates: Obtains and configures HTTPS (if domain provided)
  8. Service Creation: Creates systemd service for auto-start
  9. SSL Auto-Renewal: Sets up automatic certificate renewal

For detailed technical information, check our Advanced Installation Guide.

Troubleshooting

The installation failed. What should I do? +

First steps:

  1. Read the error message carefully - it often tells you what went wrong
  2. Check if you have sufficient permissions (run as root or with sudo)
  3. Ensure your system is updated: sudo apt update && sudo apt upgrade
  4. Try running the installer again - some issues are temporary

Common issues:

  • Port conflicts: Make sure ports 80/443 aren't already in use
  • DNS issues: Verify your domain points to your server's IP
  • Archive problems: Check that your product file isn't corrupted
  • Network issues: Ensure your server can reach the internet
# Check service status sudo systemctl status plex-PRODUCTNAME # View logs sudo journalctl -u plex-PRODUCTNAME -f # Test Nginx configuration sudo nginx -t
SSL certificate setup failed. How do I fix it? +

Common SSL issues:

  • Domain not pointing to server: Verify DNS A record
  • Ports blocked: Ensure ports 80 and 443 are open
  • Rate limits: Let's Encrypt has rate limits - wait if exceeded
  • Cloudflare proxy: Disable "Proxy status" temporarily

Manual SSL retry:

# Test certificate renewal sudo certbot renew --dry-run # Manual certificate request sudo certbot --nginx -d yourdomain.com # Check certificate status sudo certbot certificates

Note: Our installer includes SSL error recovery and will restore your Nginx config if SSL setup fails.

The service won't start. What's wrong? +

Check service status:

sudo systemctl status plex-PRODUCTNAME sudo journalctl -u plex-PRODUCTNAME --no-pager

Common causes:

  • Missing config: Product needs configuration (bot tokens, database settings)
  • Port conflicts: Another service using the same port
  • Permission issues: Files have wrong ownership/permissions
  • Missing dependencies: NPM install failed or incomplete

Manual service management:

# Start service sudo systemctl start plex-PRODUCTNAME # Stop service sudo systemctl stop plex-PRODUCTNAME # Restart service sudo systemctl restart plex-PRODUCTNAME # Enable auto-start sudo systemctl enable plex-PRODUCTNAME

Post-Installation

How do I configure my installed product? +

After installation, you'll need to configure your product:

  1. Find the config file: Usually in /var/www/plex/PRODUCTNAME/
  2. Edit configuration: Add bot tokens, database connections, etc.
  3. Restart the service: sudo systemctl restart plex-PRODUCTNAME

Common config files:

  • config.yml - YAML configuration
  • config.json - JSON configuration
  • .env - Environment variables
# Edit config with nano sudo nano /var/www/plex/PRODUCTNAME/config.yml # After editing, restart service sudo systemctl restart plex-PRODUCTNAME
How do I update an installed product? +

To update a product:

  1. Backup your config: Copy your configuration files
  2. Stop the service: sudo systemctl stop plex-PRODUCTNAME
  3. Run installer again: It will detect existing installation
  4. Choose to update: Select update option when prompted
  5. Restore config: Restore your backed-up configuration
  6. Start service: sudo systemctl start plex-PRODUCTNAME

Important: Always backup your configuration files before updating!

How do I uninstall a product? +

To remove an installed product:

# Stop and disable service sudo systemctl stop plex-PRODUCTNAME sudo systemctl disable plex-PRODUCTNAME # Remove service file sudo rm /etc/systemd/system/plex-PRODUCTNAME.service sudo systemctl daemon-reload # Remove product files sudo rm -rf /var/www/plex/PRODUCTNAME # Remove Nginx config (if web-based) sudo rm /etc/nginx/sites-available/PRODUCTNAME sudo rm /etc/nginx/sites-enabled/PRODUCTNAME sudo systemctl reload nginx # Remove SSL certificate (optional) sudo certbot delete --cert-name DOMAIN
